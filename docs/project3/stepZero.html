Step Zero

It is best practice to start with an informal scoping phase that focuses on understanding, probing the challenge, and sketching a preliminary solution, before entering structured development.

It’s where creativity meets physics and feasibility.

1. Quick Conceptualization & Brainstorming

  • Sketch rough ideas of the system configuration: where the pendulum is mounted, how the mechnical system is fixed, how the wheel is attached and rotates, where the motor and sensor go, where the wires go.

  • Discuss what the main goal is: e.g., swing-up and stabilization.

  • Look at reference examples.

2. Back-of-the-Envelope Calculations

  • Estimate:

    • Mass and length of the pendulum (\(m\), \(l\))

    • Moment of inertia of the reaction wheel (\(I\))

    • Required torque (\(\tau = I \alpha\)) during bang-bang swing-up, and during PID or LQR stabilization.

    • Approximate rotational speed ranges (\(\omega\))

  • Check if motor power, voltage, and gear ratio are within reach.

3. Quick and Dirty Prototype

  • Build a first rudimentary prototype using:

    • Readily available parts (LEGO, wood, printed parts, scrap electronics)

    • A simple DC motor and basic wheel (plastic or metal disk)

    • Temporary mounts (tape, clamps, hot glue, cardboard)

  • Power the motor manually (e.g., via a lab supply or battery) and observe the dynamics.

4. Physics Scoping and Trade-offs

  • Identify key physical quantities to control: torque, angular position, friction.

  • Recognize constraints:

    • Voltage/current limits of motors

    • Sensor resolution (e.g., AS5600 for angle)

    • Microcontroller limitations (timing, ADC resolution)

  • Think about trade-offs:

    • Larger wheel = more inertia, but heavier

    • Faster motor = higher voltage/power needs

    • Longer pendulum = slower dynamics but easier to observe

5. Sketch System Configurations

  • Draw block diagrams:

    • Mechanical layout

    • Electrical layout

    • Signal flow: Sensor → Controller → Motor → Wheel

    • Controller layout

    • Software layout

  • Identify knowns, unknowns, and to-be-measured parameters

6. Understand Project Scope and Complexity

  • Ask:

    • How many components do we need to integrate?

    • How difficult is swing-up vs. stabilization?

    • What precision do we need?

  • List major work packages:

    • Mechanics

    • Electronics

    • Control software

    • Testing

    • Documentation

7. First Draft of System Requirements Write a lightweight, flexible document that includes:

  • Objective: “Stabilize a pendulum using a reaction wheel”

  • Constraints:

    • Max motor power: 12 V, 2 A

    • Sensor: AS5600, 12-bit resolution

  • Desired behavior:

    • Swing up in less than 5 seconds

    • Stabilize within ±5°

  • Materials allowed: 3D printing, wood, aluminum, plastic

  • Tools: Arduino, basic soldering, power supply, oscilloscope
